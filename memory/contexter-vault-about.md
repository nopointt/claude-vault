## Identity

contexter-vault = local proxy that redacts secrets from Claude Code API traffic before reaching Anthropic's servers. Zero-dependency Bun process. AES-256-GCM encrypted vault stored at `~/.contexter-vault/`. Open-source MIT.

Renamed from `claude-vault` in V-01 (2026-04-21). First official open-source release.

## Problem It Solves

Claude Code sends every message (including typed API keys, tokens, credentials) to Anthropic's API as conversation context. Transcripts shared via `/feedback` retained up to 5 years. No built-in redaction. contexter-vault intercepts traffic, replaces secret values with `<<VAULT:name>>` placeholders, restores real values only at local tool execution time.

## Architecture — Four Layers

| Layer | Mechanism | Purpose |
|---|---|---|
| 1. Proxy | HTTP server on 127.0.0.1:9277, `ANTHROPIC_BASE_URL` points here | Scans request body + SSE response stream, replaces secrets ↔ placeholders |
| 2. UserPromptSubmit hook | `src/hooks/secret-store.ts` | Handles `/secret store <name>` — reads value from `~/.contexter-vault/buffer.txt`, encrypts, wipes buffer |
| 3. PreToolUse hook | `src/hooks/pre-tool-use.ts` | Substitutes `<<VAULT:name>>` with real values in Bash/Write/Edit commands at execution time |
| 4. .claudeignore | Generated by `init` | Blocks Claude Read tool from `.contexter-vault/`, `*.key`, `*.enc` |

## CLI Commands

| Command | Purpose |
|---|---|
| `contexter-vault init` | Create vault, generate 256-bit key, set `ANTHROPIC_BASE_URL`, register hooks in settings.json, seed `.claudeignore` |
| `contexter-vault start` | Start proxy on 127.0.0.1:9277 (in-process, PID in `~/.contexter-vault/proxy.pid`) |
| `contexter-vault stop` | Kill proxy PID from `.pid` file |
| `contexter-vault add <name>` | Read from `~/.contexter-vault/buffer.txt` OR stdin, encrypt, store, wipe buffer |
| `contexter-vault remove <name>` | Delete secret from vault |
| `contexter-vault list` | List secret names with masked preview (first 4 chars + asterisks) |
| `contexter-vault status` | Show vault path, secret count, proxy PID state |

## Encryption

- Algorithm: AES-256-GCM
- Key: 256-bit random, stored plaintext at `~/.contexter-vault/vault.key` (file perms only, no OS-level secure enclave in V-01)
- IV: 12 bytes per write (fresh per encryption)
- Auth tag: 16 bytes, prevents tampering
- Storage format: `iv (12 bytes) || ciphertext || auth_tag (16 bytes)` — no version header (B-13 flagged for V-03)

## Proxy Flow

```
POST /v1/messages (streaming):
  1. Read request body as text
  2. Load secrets from vault (cached 5s TTL)
  3. redactString(body, secrets) — replace all secret values with placeholders
  4. fetch(Anthropic API, body=redactedBody, headers=passthrough)
  5. If streaming: ReadableStream with pull()
     - Reads upstream chunks, buffers sseBuffer
     - When buffer > maxSecretLen: redact entire buffer, emit safe prefix, keep maxSecretLen tail
     - On done: flush remaining buffer
     - On error: try/catch, flush, emit synthetic SSE error event, controller.close()
  6. If non-streaming: read full response text, redact, return

POST /v1/messages/count_tokens:
  1. Redact body, forward, redact response. No streaming.

Other paths: 404
```

**SSE Sliding window algorithm:** retains `maxSecretLen` bytes as tail carryover between chunks. Handles secrets split across TCP packet boundaries. Covered by `src/test-redaction.ts` (with B-01 tautology bug — deferred to V-02).

## Security Model

### Protected
- Secret values in user messages, redacted before reaching Anthropic
- Secret values in Claude SSE responses, redacted
- Vault file AES-256-GCM encrypted on disk
- Buffer file wiped immediately after reading (non-secure wipe — B-14)

### NOT Protected
- Secrets in Claude Code's local JSONL conversation logs (`~/.claude/projects/`)
- Secrets echoed to stdout by tool runs — appear in tool_result of current turn (redacted on next API call, but one turn may leak)
- Vault key file plaintext on disk — protected by filesystem perms only (chmod 600 not enforced in V-01 — B-15)
- Secrets visible in non-ANTHROPIC_BASE_URL traffic (web requests, other tools)

### Compliance
- `ANTHROPIC_BASE_URL` is officially documented LLM gateway mechanism
- Hooks (UserPromptSubmit, PreToolUse) are officially documented extension system
- All auth headers (`anthropic-version`, `x-api-key`, `anthropic-beta`, `X-Claude-Code-Session-Id`) forwarded unchanged
- Usage fully attributable to user's Anthropic account
- Not a gateway that batches/deduplicates/rewrites — purely transparent passthrough with redaction

## Integration With Claude Code

Two touch points in `~/.claude/settings.json`:

1. **env.ANTHROPIC_BASE_URL** = `http://127.0.0.1:9277` — seamless interception
2. **hooks.UserPromptSubmit** + **hooks.PreToolUse** — register hook commands pointing to `src/hooks/*.ts` via bun run

One touch point in `~/.claudeignore`:
- `.contexter-vault/`, `*.key`, `*.enc` — prevent Claude Read tool from direct access to vault files

All three managed by `contexter-vault init`.
